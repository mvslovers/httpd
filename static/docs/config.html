HTTPD Documentation

Configuration

Default Settings

Starting with the HTTPD 3.0.0 release the HTTPD server uses a configuration file (dataset or PDS member) written in the Lua language.

The following list of settings are from the 3.0.0 HTTPD release. As new HTTPD settings are created this list may be less than the actual number of settings possible. We'll show you how to see all possible settings when we get to using Lua in the next section.

httpd.debug="0"
This setting enables or disables debug output to the dataset allocated to DD HTTPDBG. 0 is disabled. 1 is enabled.
httpd.bind_tries="10"
Number of attempts the server will make to bind the listener socket for the HTTPD server.
httpd.bind_sleep="10"
Number of seconds the server will sleep between failed attempts to bind the listener socket for the HTTPD server.
httpd.client_stats="1"
This setting enables or disables statistics gathering related to client processing. When enabled statistics are recorded for each response code and processing duration. 0 is disabled. 1 is enabled.
httpd.client_stats_month_max="24"
Maximum number of statistic records to be collected per month. Each record holds data for a month+response pair.
httpd.client_stats_day_max="60"
Maximum number of statistic records to be collected per day. Each record holds data for a day+response pair.
httpd.client_stats_hour_max="48"
Maximum number of statistic records to be collected per hour. Each record holds data for a hour+response code pair.
httpd.client_stats_minute_max="48"
Maximum number of statistic records to be collected per minute. Each record holds data for a minute+response code pair.
httpd.client_stats_dataset=""
Name of dataset used to save and load statistics records at HTTPD server startup and shutdown. If an empty string is used, no statistics will be saved or loaded by the server. If the dataset doesn't exist it will be created with RECFM=PS,LRECL=80,BLKSIZE=27920,SPACE=(TRK,(1,1)).
httpd.client_timeout="10"
Sets the time the server will wait for a client to send a complete request to the server after the client has connected to the server. After this timeout seconds expire the server will close the connection without any further processing.
httpd.client_timeout_msg="1"
Enables or disables console messages related to a client connection timeout occuring. 0 is disabled. 1 is enabled.
httpd.client_timeout_dump="0"
Enables or disables console dump of the clients buffer when a timeout occurs. 0 is disabled. 1 is enabled.
httpd.ftp="1"
Enables or disables FTPD as part of the HTTPD server. 0 is disabled. 1 is enabled.
httpd.listen_queue="5"
Number of listen queue sockets for incomming client connections.
httpd.login="none"
Login requirement for various types of request. the login string can be one or more of these values "all,cgi,get,head,post,none". The default is "none" meaning that all request are allowed without a login.
httpd.mintask="3"
Sets the minimum number of task (threads) that the server should keep available for servicing client request.
httpd.maxtask="9"
Sets the maximum number of task (threads) that the server will start to service client request.
httpd.tzoffset="0"
Sets the time zone offset in minutes East (negative number) or West (positive number) of GMT. For example in my time zone (Central Time USA) I use httpd.tzoffset="-300" which is GMT -5 Hours.
httpd.port="8080"
This is the port number the HTTPD server will listen on for client (Web Browser) request.
httpd.ufs="1"
Enables or disables the Unix File System (UFS) in the HTTPD server. 0 is disabled. 1 is enabled.
cgi.program="path/pattern"
Each program="path/pattern" represents the program that the HTTPD server will link to whenever the request URL matches the "path/pattern". Example: 
cgi.HTTPREXX="/rexx/*"
cgi.ABENDOC1="/abend0c1"
cgi.HELLO="/hello"
cgi.HTTPLUA="/lua/*"
cgi.HTTPJES2="/jes/*"
cgi.HTTPDM="/.dm"
cgi.HTTPDMTT="/.dmtt"
cgi.HTTPDSRV="/.dsrv"
ftpd.bind_tries="10"
Number of attempts the server will make to bind the listener socket for the FTPD server.
ftpd.bind_sleep="10"
Number of seconds the server will sleep between failed attempts to bind the listener socket for the FTPD server.
ftpd.listen_queue="5"
Number of listen queue sockets for incomming client connections.
ftpd.port="8021"
This is the port number the FTPD server will listen on for FTP client request.

In HTTPD 3.1.0 we got these new settings:

httpd.cgilua_dataset="HTTPD.LUA"
specifies the dataset name used when a CGI Lua script is executed. The dataset should be a PDS type dataset with attributes like: RECFM=VB,LRECL=255,BLKSIZE=27920,DSORG=PO
Your CGI Lua script members should reside in this dataset.
httpd.cgilua_path=""
specifies the Lua package.path names that will replace the default package.path names provided by LUA370 code. The package.path is what Lua uses when it encounters a require'something' statement in which something is the Lua member name. 
Example: 
    httpd.cgilua_path="HERC01.HTTPD.LUA(?);HTTPD.LUA(?)"
Givin the above example and a require'http' statement in your Lua script, Lua will look for 'http' in HERC01.HTTPD.LUA(http), then in HTTPD.LUA(http) and if still not found, issue an error.

If you want to keep the LUA370 defaults you could specify httpd.cgilua_path="HERC01.HTTPD.LUA(?);HTTPD.LUA(?);"..package.path in your configuration file.

Keep in mind that you can also change the package.path string at runtime prior to any require statements being used in your Lua script.

Default LUA370 package.path values:
The default Lua package.path in TSO (userid HERC01) with PROFILE PREFIX(MDR) is: MDR.LUA370.LUA(?);MDR.LUA(?);HERC01.LUA370.LUA(?);HERC01.LUA(?); LUA370.LUA(?);LUA370.LOCAL.LUA(?);LUA370.SHARE.LUA(?)

The default Lua package.path in TSO (userid HERC01) with PROFILE NOPREFIX is: HERC01.LUA370.LUA(?);HERC01.LUA(?);LUA370.LUA(?);LUA370.LOCAL.LUA(?);LUA370.SHARE.LUA(?)

The default Lua package.path in BATCH (userid HERC01) is: HERC01.LUA370.LUA(?);HERC01.LUA(?);LUA370.LUA(?);LUA370.LOCAL.LUA(?);LUA370.SHARE.LUA(?)

The default Lua package.path in STC (started task) is: STC.LUA370.LUA(?);STC.LUA(?);LUA370.LUA(?);LUA370.LOCAL.LUA(?);LUA370.SHARE.LUA(?)

httpd.cgilua_cpath=""
specifies the Lua package.cpath names that will replace the default package.cpath names provided by LUA370 code. The package.cpath is what Lua uses when it encounters a require'something' in which something is the member name.

Note package.cpath is for "compiled" Lua scripts which we're not presently doing. I still need to work out any issues (I haven't done any testing yet) in LUA370 and compiled Lua scripts.

In HTTPD 3.2.0 we got these new settings:

mqtt.start="0"
a non zero value enables the Message Queuing Telemetry Transport (MQTT) client which will then attempt to connect to a MQTT Broker so that the HTTPD server can publish topics which represent logical components of the HTTPD server. The default "0" setting disables MQTT in the HTTPD server.
mqtt.broker_host="test.mosquitto.org"
specifies the host name of the MQTT Broker (server). If you have your own MQTT Broker (server) you should specify the host name or the IP address in the mqtt.broker_host setting.
mqtt.broker_port="1883"
specifies the port number of the MQTT Broker (server). If your mqtt.broker_host is listening on a non stadard port then you should specify that port number in the mqtt.broker_port setting.
mqtt.prefix=""
specifies the topic prefix to be used when the HTTPD server publishes a new topic. If mqtt.prefix is "" then a default prefix will be created as "smfid/jobname" like "TK5R/HTTPD".
Note: The topic prefix should NOT start with a "/" character.
mqtt.datetime_gmt="0"
a non zero value enables "datetime" values in the published topics to be GMT values. Otherwise the "datetime" values will be in local time.
mqtt.telemetry="30"
a non zero value enables the telemetry thread and represents the inteval at which the telemetry thread will publish topics for values that change while the HTTPD server is running.

In HTTPD 3.2.2 we got these new settings:

httpd.cgi_context_pointers="255"
number of CGI context pointers to be allocated in the HTTPD structure. The CGI context pointers are for use by CGI applications that need to anchor storage in the HTTPD address space. The default allocation is 255 void pointers are initialized to 0 by the HTTPD server at startup.

Configure HTTPD Using Lua

A configuration file can be specified via the PARM='CONFIG=your.config.dataset' on the EXEC card for PGM=HTTPD.

//HTTPD    EXEC PGM=HTTPD,REGION=8000K,TIME=1440,
//         PARM='CONFIG=HTTPD.LUA(CONFIG)'

At HTTPD startup the server prepares a Lua environment with several Lua tables (httpd, cgi, ftpd) with the default server settings. See the list above.

If the default values are acceptable for your site then you don't need any PARM=... setting at all. You only need a HTTPD configuration file if you want to change one or more settings or if you want to examine the configuration settings.

A Write To Operator function has been provided in the Lua configuration environment as the global "wto". Example: wto('Hello World')

Example: My configuration file (comments start with -- ): 

  -- Lua HTTPD Configuration
  -- httpd.login='(cgi,get)'
  httpd.port=1080
  httpd.tzoffset=-300
  httpd.client_timeout=1
  httpd.client_timeout_msg=1
  httpd.client_timeout_dump=0
  httpd.client_stats_dataset='MDR.HTTPD.STATS'
  httpd.ftp=1
  ftpd.port=2021
  mqtt.start="1"
  mqtt.prefix="tk5/httptest"
  mqtt.telemetry="10"
  mqtt.broker_host="mosquitto.rayborn.us"
  -- Dump it all to the console
  wto('HTTPD Configuration')
  -- wto('Global Variables')
  -- for n in pairs(_G) do wto(n) end
  -- wto('-------------------------------')
  wto('HTTPD Table')
  for n,v in pairs(httpd) do wto('httpd.'..n..'='..'"'..v..'"') end
  wto('-------------------------------')
  wto('CGI Table')
  for n,v in pairs(cgi) do wto('cgi.'..n..'='..'"'..v..'"') end
  wto('-------------------------------')
  wto('FTPD Table')
  for n,v in pairs(ftpd) do wto('ftpd.'..n..'='..'"'..v..'"') end
  wto('-------------------------------')
  wto('MQTT Table')
  for n,v in pairs(mqtt) do wto('mqtt.'..n..'='..'"'..v..'"') end
If all you want is a complete list of setting defaults you can place the following in the configuration file before you make any other changes to the settings: 
  wto('HTTPD Configuration Defaults')
  wto('-------------------------------')
  wto('HTTPD Table')
  for n,v in pairs(httpd) do wto('httpd.'..n..'='..'"'..v..'"') end
  wto('-------------------------------')
  wto('CGI Table')
  for n,v in pairs(cgi) do wto('cgi.'..n..'='..'"'..v..'"') end
  wto('-------------------------------')
  wto('FTPD Table')
  for n,v in pairs(ftpd) do wto('ftpd.'..n..'='..'"'..v..'"') end
  wto('-------------------------------')
  wto('MQTT Table')
  for n,v in pairs(mqtt) do wto('mqtt.'..n..'='..'"'..v..'"') end
Restart the HTTPD server (stop and then start HTTPD) with a PARM='CONFIG=my.config.dataset(member)' setting on the HTTPD EXEC card. Then view the output on the MVS system console.

If you discover a setting that's not documented above you will likely find those in the changelog file included in the zip file for this HTTPD release.
